# Отладка изменения скидки для Сбербанка

## Проблема

При изменении скидки с 30% на 40% в конструкторе для Сбербанка база не пересчитывается.

## Добавлена расширенная отладка

В консоли браузера теперь выводится детальная информация:

```
=== REFRESH START ===
prevInsAmount: 3648641.81
prevDiscountPercent: 30
Сбербанк: значение из input: 40
Сбербанк: новая скидка после clamp: 40
Проверка изменений: {
  ins: 3648641.81,
  prevInsAmount: 3648641.81,
  insChanged: false,
  prevDiscountPercent: 30,
  currentDiscountPercent: 40,
  discountChanged: true,
  needRecalcBase: true
}
Пересчитываем базу
```

## Инструкции по тестированию

1. **Переустановите расширение** (версия 1.0.4)
2. **Откройте консоль браузера** (F12 → Console)
3. **Выполните расчет для Сбербанка**:
   ```
   Сбербанк 15.01.2024
   осз 5 000 000
   муж. 01.01.1985
   квартира гп 2020
   ```
4. **Откройте конструктор варианта 2**
5. **Измените скидку с 30% на 40%**
6. **Проверьте консоль**:
   - Должно быть `prevDiscountPercent: 30`
   - Должно быть `currentDiscountPercent: 40`
   - Должно быть `discountChanged: true`
   - Должно быть `needRecalcBase: true`
   - Должно быть `Пересчитываем базу`

## Что проверить

1. **База пересчитывается** при изменении скидки
2. **База НЕ пересчитывается** при изменении галочек доп. рисков
3. **База пересчитывается** при изменении страховой суммы

## Если проблема сохраняется

Пришлите скриншот консоли с сообщениями отладки. Это поможет понять:
- Правильно ли сохраняется `prevDiscountPercent`
- Правильно ли обновляется `state.discountPercent`
- Срабатывает ли условие `discountChanged`

## Возможные причины

1. **Input не обновляется** - значение из `discountInput.value` не меняется
2. **clampDiscountPercent возвращает старое значение** - функция работает неправильно
3. **state.discountPercent не обновляется** - проблема с объектом state
4. **Событие не срабатывает** - обработчик `input`/`change` не вызывается
