# Исправление сохранения скидки при смене банка (КРИТИЧЕСКОЕ)

## Проблема

После расчета для Сбербанка со скидкой 40% в конструкторе, при расчете для другого банка (например, ВТБ) эта скидка 40% применялась к базовым рискам нового расчета.

**Пример**:
1. Рассчитали Сбербанк → открыли конструктор → установили скидку 40%
2. Закрыли конструктор (база пересчиталась и сохранилась в `ctx.variant2Meta.base`)
3. Рассчитали ВТБ → открыли конструктор → база использует скидку 40% (неправильно!)
4. ВТБ должен иметь фиксированную скидку 30%

## Причина

`ctx.variant2Meta.base` сохранялся между расчетами и содержал базу, рассчитанную со скидкой 40% для Сбербанка.

При открытии конструктора для ВТБ:
```javascript
if (!ctx.variant2Meta.base) {
  // Инициализируем базу
  ctx.variant2Meta.base = computeVariant2BasePremiums(...);
}
```

Но `ctx.variant2Meta.base` уже существовал (от Сбербанка со скидкой 40%), поэтому не пересчитывался!

## Решение

При смене банка сбрасываем не только `ctx.variant2CustomState`, но и `ctx.variant2Meta.base`:

```javascript
const savedBankName = ctx.variant2CustomState?.bankName;
const currentBankName = ctx.bankConfig?.bankName;
if (savedBankName && savedBankName !== currentBankName) {
  console.log('Банк изменился с', savedBankName, 'на', currentBankName, '- сбрасываем состояние и базу');
  ctx.variant2CustomState = null;
  // КРИТИЧНО: также сбрасываем базу, т.к. она была рассчитана со скидкой старого банка
  if (ctx.variant2Meta) {
    ctx.variant2Meta.base = null;
  }
}
```

## Тест

### Шаг 1: Расчет для Сбербанка со скидкой 40%

```
Сбербанк 15.01.2024
осз 5 000 000
муж. 01.01.1985
квартира гп 2020
```

1. Открываем конструктор
2. Меняем скидку на 40%
3. Проверяем базу (должна быть меньше чем с 30%)
4. Закрываем конструктор

### Шаг 2: Расчет для ВТБ

```
ВТБ 31.03.2023
осз 6 161 736
муж.03.05.1981
квартира гп 2017
```

1. Открываем конструктор
2. **В консоли**: должно быть "Банк изменился с Сбербанк на ВТБ - сбрасываем состояние и базу"
3. **Проверяем базу**: должна быть 17 317,56 ₽ (с фиксированной скидкой 30%)
4. **НЕ должна быть**: база со скидкой 40% от Сбербанка

### Шаг 3: Проверка что база не меняется при изменении галочек

1. Добавляем доп. риск (отделка 200 000)
2. **Проверяем**: база остается 17 317,56 ₽
3. Убираем галочку
4. **Проверяем**: база остается 17 317,56 ₽

## Ключевые моменты

### Что сохраняется между открытиями конструктора

**В рамках ОДНОГО расчета** (один банк):
- `ctx.variant2CustomState` - настройки UI (галочки, суммы, скидка)
- `ctx.variant2Meta.base` - рассчитанная база (имущество + жизнь + титул)

**При НОВОМ расчете** (другой банк):
- `ctx.variant2CustomState` сбрасывается → `null`
- `ctx.variant2Meta.base` сбрасывается → `null`
- Оба пересчитываются заново с правильными параметрами нового банка

### Почему это критично

Если не сбрасывать `ctx.variant2Meta.base`:
- ВТБ будет использовать базу со скидкой 40% (от Сбербанка)
- Вместо правильной базы со скидкой 30%
- Итоговая сумма варианта 2 будет неправильной
- Клиент получит неверный расчет
