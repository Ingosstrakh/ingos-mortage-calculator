# Исправление: Привязка контекста к кнопке конструктора

## Проблема

После нажатия "Очистить предыдущие" проблема со скидкой все равно сохранялась. Из логов видно:

```
1. Открывается конструктор для ВТБ: isSberbank: false ✅
2. Снова открывается для Сбербанка: isSberbank: true, prevDiscountPercent: 50 ❌
```

**Причина**: Когда на странице несколько результатов расчета, у каждого есть кнопка "Конструктор варианта 2". Все кнопки вызывают `window.openVariant2Constructor()` БЕЗ параметров, что означает использование ГЛОБАЛЬНОГО контекста `window.__LAST_VARIANT2_CONTEXT__`.

Даже после очистки старых результатов, если случайно прокрутить страницу вверх и нажать старую кнопку, откроется конструктор со старыми данными.

## Решение

Каждая кнопка "Конструктор варианта 2" теперь хранит СВОЙ контекст и передает его в функцию.

### Было (неправильно):
```javascript
// Все кнопки используют один глобальный контекст
output += `<button onclick="window.openVariant2Constructor()">
  Конструктор варианта 2
</button>`;

// Функция всегда берет глобальный контекст
window.openVariant2Constructor = function() {
  const ctx = window.__LAST_VARIANT2_CONTEXT__; // ❌ Может быть старым!
  // ...
}
```

### Стало (правильно):
```javascript
// Каждая кнопка хранит СВОЙ контекст с уникальным ID
const contextId = 'ctx_' + Date.now();
window['__VARIANT2_CONTEXT_' + contextId] = {
  parsedData: data,
  bankConfig: bankConfig,
  // ... все данные конкретного расчета
};

output += `<button onclick="window.openVariant2Constructor(window['__VARIANT2_CONTEXT_${contextId}'])">
  Конструктор варианта 2
</button>`;

// Функция принимает контекст как параметр
window.openVariant2Constructor = function(forceContext = null) {
  const ctx = forceContext || window.__LAST_VARIANT2_CONTEXT__; // ✅ Используем переданный контекст
  // ...
}
```

## Как это работает

### 1. При создании результата расчета
```javascript
// Генерируем уникальный ID для контекста
const contextId = 'ctx_1709654321000'; // timestamp

// Сохраняем контекст в глобальной переменной с уникальным именем
window['__VARIANT2_CONTEXT_ctx_1709654321000'] = {
  parsedData: { bank: 'ВТБ', osz: 6000000, ... },
  bankConfig: { bankName: 'ВТБ', ... },
  variant1Total: 83226,
  insuranceAmount: 6600000,
  variant2Meta: { ... },
  variant2CustomState: null
};

// Кнопка вызывает функцию с ЭТИМ контекстом
<button onclick="window.openVariant2Constructor(window['__VARIANT2_CONTEXT_ctx_1709654321000'])">
```

### 2. При нажатии кнопки
```javascript
// Функция получает контекст конкретного расчета
window.openVariant2Constructor(window['__VARIANT2_CONTEXT_ctx_1709654321000'])

// Внутри функции:
const ctx = forceContext || window.__LAST_VARIANT2_CONTEXT__;
// ctx теперь содержит данные ИМЕННО ЭТОГО расчета (ВТБ)
// а не глобального контекста (который может быть от Сбербанка)
```

### 3. Результат
- Каждая кнопка "привязана" к своему расчету
- Невозможно случайно открыть конструктор с данными другого банка
- Даже если на странице 10 результатов, каждый работает независимо

## Преимущества

### ✅ Изолированные контексты
Каждый результат расчета имеет свой контекст, который не перезаписывается при новых расчетах.

### ✅ Защита от ошибок
Даже если пользователь не нажал "Очистить предыдущие", каждая кнопка открывает конструктор с правильными данными.

### ✅ Множественные результаты
Можно иметь на странице результаты для разных банков и работать с каждым независимо.

### ✅ Обратная совместимость
Если `forceContext` не передан, функция использует глобальный контекст (как раньше).

## Workflow

### Сценарий 1: Несколько расчетов подряд
```
1. Рассчитали Сбербанк (скидка 50%)
   → Создан контекст ctx_1000 для Сбербанка
   
2. Рассчитали ВТБ
   → Создан контекст ctx_2000 для ВТБ
   
3. Нажали "Конструктор" у ВТБ
   → Открывается с ctx_2000 (ВТБ) ✅
   
4. Случайно нажали "Конструктор" у Сбербанка
   → Открывается с ctx_1000 (Сбербанк со скидкой 50%) ✅
   → Но это ПРАВИЛЬНО, т.к. это данные Сбербанка!
```

### Сценарий 2: С очисткой предыдущих
```
1. Рассчитали Сбербанк
2. Рассчитали ВТБ
3. Нажали "Очистить предыдущие"
   → Удален результат Сбербанка со страницы
   → Остался только ВТБ
4. Нажали "Конструктор"
   → Открывается с контекстом ВТБ ✅
```

## Технические детали

### Генерация уникального ID
```javascript
const contextId = 'ctx_' + Date.now();
// Пример: 'ctx_1709654321000'
```

### Сохранение контекста
```javascript
window['__VARIANT2_CONTEXT_' + contextId] = {
  parsedData: JSON.parse(JSON.stringify(data)), // Deep copy
  bankConfig: JSON.parse(JSON.stringify(bankConfig)),
  variant1Total: totalWithoutDiscount,
  insuranceAmount: insuranceAmount,
  variant2Meta: variant2Result.meta || null,
  variant2CustomState: null
};
```

### Передача контекста в кнопку
```javascript
onclick="window.openVariant2Constructor(window['__VARIANT2_CONTEXT_${contextId}'])"
```

### Функция с параметром
```javascript
window.openVariant2Constructor = function openVariant2Constructor(forceContext = null) {
  // Если передан forceContext, используем его
  // Иначе используем глобальный (для обратной совместимости)
  const ctx = forceContext || window.__LAST_VARIANT2_CONTEXT__;
  // ...
}
```
