# ФИНАЛЬНОЕ исправление скидки между банками

## Проблема (критическая)

После расчета Сбербанка со скидкой 40% в конструкторе, ВСЕ последующие расчеты (ВТБ, Дом.РФ и т.д.) использовали эту скидку 40% вместо стандартной 30%.

**Даже кнопка "Сбросить" не помогала!**

## Причина

Проблема была в нескольких местах:

### 1. `ctx.variant2Meta.base` сохранялся между расчетами
При изменении скидки в конструкторе:
```javascript
ctx.variant2Meta.base = baseNow; // Сохраняется база со скидкой 40%
```

При следующем расчете (ВТБ):
```javascript
if (!ctx.variant2Meta.base) {
  // Инициализируем базу
} // НЕ выполняется, т.к. base уже существует!
```

### 2. Проверка смены банка не срабатывала
Проверка была только в `openVariant2Constructor()`, но:
- Если не открывать конструктор, проверка не срабатывает
- База уже использовалась в `calculateVariant2()` ДО открытия конструктора

### 3. Глобальный контекст не очищался
`window.__LAST_VARIANT2_CONTEXT__` сохранялся между расчетами и содержал:
- `variant2Meta.base` со скидкой 40%
- `variant2CustomState` с настройками Сбербанка

## Решение

### 1. Добавлена отладка в начале performCalculations

```javascript
function performCalculations(data) {
  // КРИТИЧНО: Очищаем старый контекст при новом расчете
  const oldContext = window.__LAST_VARIANT2_CONTEXT__;
  const oldBankName = oldContext?.bankConfig?.bankName;
  console.log('Новый расчет. Старый банк:', oldBankName, 'Новый банк:', data.bank);
  
  // ... остальной код
}
```

### 2. Сброс базы при смене банка в конструкторе

```javascript
if (savedBankName && savedBankName !== currentBankName) {
  console.log('Банк изменился - сбрасываем состояние и базу');
  ctx.variant2CustomState = null;
  if (ctx.variant2Meta) {
    ctx.variant2Meta.base = null; // КРИТИЧНО!
  }
}
```

### 3. Принудительная установка null для не-Сбербанка

```javascript
if (!isSberbank) {
  state.discountPercent = null;
}
```

## Полный тест

### Шаг 1: Сбербанк со скидкой 40%

```
Сбербанк 15.01.2024
осз 5 000 000
муж. 01.01.1985
квартира гп 2020
```

1. Выполняем расчет
2. **В консоли**: "Новый расчет. Старый банк: undefined Новый банк: Сбербанк"
3. Открываем конструктор
4. Меняем скидку на 40%
5. **Проверяем базу**: должна уменьшиться (скидка 40% вместо 30%)
6. Закрываем конструктор

### Шаг 2: ВТБ (проверка сброса)

```
ВТБ 31.03.2023
осз 6 161 736
муж.03.05.1981
квартира гп 2017
```

1. Выполняем расчет
2. **В консоли**: "Новый расчет. Старый банк: Сбербанк Новый банк: ВТБ"
3. **Проверяем вариант 2 БЕЗ открытия конструктора**: база должна быть со скидкой 30%
4. Открываем конструктор
5. **В консоли**: "Банк изменился с Сбербанк на ВТБ - сбрасываем состояние и базу"
6. **Проверяем базу**: 17 317,56 ₽ (со скидкой 30%, НЕ 40%)

### Шаг 3: Дом.РФ (еще одна проверка)

```
Дом.РФ 23.07.2025
осз 3 648 641
дом дерево/дерево 2025
```

1. Выполняем расчет
2. **В консоли**: "Новый расчет. Старый банк: ВТБ Новый банк: Дом.РФ"
3. **Проверяем вариант 2**: база должна быть БЕЗ скидки (Дом.РФ не разрешает скидки)

### Шаг 4: Снова Сбербанк

```
Сбербанк 20.02.2024
осз 4 000 000
жен. 15.03.1990
квартира
```

1. Выполняем расчет
2. **В консоли**: "Новый расчет. Старый банк: Дом.РФ Новый банк: Сбербанк"
3. Открываем конструктор
4. **Проверяем скидку**: должна быть 30% (по умолчанию, НЕ 40% от предыдущего Сбербанка)

## Ключевые моменты

### Что должно происходить

1. **При каждом новом расчете** выводится "Новый расчет. Старый банк: X Новый банк: Y"
2. **При смене банка** в конструкторе выводится "Банк изменился - сбрасываем состояние и базу"
3. **База пересчитывается** с правильной скидкой для нового банка
4. **Кнопка "Сбросить"** в конструкторе сбрасывает настройки к начальным значениям

### Что НЕ должно происходить

1. ❌ Скидка от Сбербанка НЕ должна применяться к другим банкам
2. ❌ База НЕ должна сохраняться между расчетами разных банков
3. ❌ Настройки конструктора НЕ должны переноситься на другой банк

## Инструкции

1. **ОБЯЗАТЕЛЬНО переустановите расширение** (версия 1.0.9)
2. **Откройте консоль браузера** (F12 → Console)
3. **Выполните полный тест** (все 4 шага)
4. **Проверьте консоль** на каждом шаге
5. **Убедитесь** что скидка НЕ переносится между банками

## Если проблема сохраняется

Пришлите скриншот консоли со всеми сообщениями. Должно быть:
- "Новый расчет. Старый банк: ... Новый банк: ..."
- "Банк изменился с ... на ... - сбрасываем состояние и базу"
- Значения базы для каждого банка
