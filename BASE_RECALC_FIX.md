# Исправление пересчета базы при изменении доп. рисков

## Проблема

База (имущество + жизнь + титул) менялась при добавлении/удалении доп. рисков:
- Без доп. рисков: база 7 353,29 ₽
- С доп. риском: база 8 964,97 ₽ (неправильно!)

База должна меняться ТОЛЬКО при изменении:
1. Страховой суммы
2. Процента скидки (только для Сбербанка)

## Причина

В функции `refresh()` условие `needRecalcBase` работало неправильно:

```javascript
const needRecalcBase = (ins !== prevInsAmount) || (state.discountPercent !== prevDiscountPercent);
```

Проблемы:
1. При первом вызове `prevInsAmount` был `undefined`, поэтому `ins !== undefined` всегда `true`
2. Строгое сравнение `!==` не учитывало погрешности округления

## Решение

Изменена логика проверки:

```javascript
// Проверяем изменение страховой суммы (с учетом округления)
const insChanged = Math.abs(ins - (prevInsAmount || ins)) > 0.01;

// Проверяем изменение скидки (для не-Сбербанка оба null, поэтому не изменилось)
const discountChanged = prevDiscountPercent !== state.discountPercent;

const needRecalcBase = insChanged || discountChanged;
```

**Ключевые изменения**:
1. `prevInsAmount || ins` - если `prevInsAmount` undefined, используем текущее значение `ins`
2. `Math.abs(...) > 0.01` - проверка с учетом погрешности округления
3. Добавлены console.log для отладки

## Добавлена отладка

В консоли браузера теперь выводится:
```
Refresh: {
  ins: 3648641.81,
  prevInsAmount: 3648641.81,
  insChanged: false,
  prevDiscountPercent: null,
  currentDiscountPercent: null,
  discountChanged: false,
  needRecalcBase: false
}
Используем сохраненную базу
База: { propertyPremiumV2: 7353.29, lifePremiumV2: 0, titlePremiumV2: 0 }
```

## Тест

**Входные данные** (любой банк кроме Сбербанка):
```
ВТБ 31.03.2023
осз 6 161 736
муж.03.05.1981
квартира гп 2017
```

**Ожидаемое поведение**:
1. Открываем конструктор - база 17 317,56 ₽
2. Добавляем доп. риск (отделка 200 000) - база остается 17 317,56 ₽
3. Убираем доп. риск - база остается 17 317,56 ₽
4. Меняем страховую сумму - база пересчитывается
5. Меняем скидку (только Сбербанк) - база пересчитывается

## Файлы изменены

1. `calculator-variant2-ui.js` - исправлена логика в функции `refresh()`
2. `extension/calculator-variant2-ui.js` - копия для расширения
3. `extension/manifest.json` - версия обновлена до `1.0.3`

## Инструкции по обновлению

1. Удалите старое расширение
2. Установите новую версию из папки `extension`
3. Проверьте версию `1.0.3` в списке расширений
4. Откройте консоль браузера (F12) для просмотра отладочных сообщений
