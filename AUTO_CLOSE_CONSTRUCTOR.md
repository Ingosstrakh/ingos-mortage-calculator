# Автоматическое закрытие конструктора при новом расчете

## Проблема (найдена в консоли)

Из консоли видно:
```
1. Рассчитали ВТБ → открыли конструктор
   isSberbank: false, prevDiscountPercent: null ✅

2. НЕ ЗАКРЫЛИ конструктор, рассчитали Сбербанк
   isSberbank: true, prevDiscountPercent: 50 ❌
```

**Конструктор остался открытым** и обновился для нового расчета (Сбербанк), используя СТАРЫЙ контекст с настройками от предыдущего Сбербанка (скидка 50%)!

## Причина

При новом расчете:
1. Создается новый контекст `window.__LAST_VARIANT2_CONTEXT__`
2. НО конструктор (модальное окно) остается открытым
3. Конструктор использует ГЛОБАЛЬНЫЙ контекст `ctx`
4. Если конструктор открыт, он продолжает работать со СТАРЫМ контекстом

## Решение

Автоматически закрываем конструктор при каждом новом расчете:

```javascript
function performCalculations(data) {
  // КРИТИЧНО: Закрываем конструктор если он открыт
  if (typeof window.closeVariant2Constructor === 'function') {
    window.closeVariant2Constructor();
  }
  
  // ... остальной код
}
```

## Что изменилось

### До исправления
1. Открыли конструктор для Сбербанка → установили скидку 50%
2. НЕ закрыли конструктор
3. Сделали расчет для ВТБ
4. Конструктор остался открытым и показывал настройки Сбербанка
5. При открытии конструктора для ВТБ использовались настройки Сбербанка

### После исправления
1. Открыли конструктор для Сбербанка → установили скидку 50%
2. Делаем расчет для ВТБ
3. **Конструктор автоматически закрывается**
4. При открытии конструктора для ВТБ используются правильные настройки ВТБ

## Тест

### Шаг 1: Сбербанк со скидкой 50%

```
Сбербанк 15.01.2024
осз 6 000 000
муж. 01.01.1985
квартира гп 2020
```

1. Выполняем расчет
2. Открываем конструктор
3. Меняем скидку на 50%
4. **НЕ ЗАКРЫВАЕМ конструктор вручную**

### Шаг 2: ВТБ (конструктор должен закрыться)

```
ВТБ 31.03.2023
осз 6 161 736
муж.03.05.1981
квартира гп 2017
```

1. Выполняем расчет
2. **Проверяем**: конструктор должен автоматически закрыться
3. **В консоли**: "Новый расчет. Старый банк: Сбербанк Новый банк: ВТБ"
4. Открываем конструктор заново
5. **В консоли**: "isSberbank: false", "prevDiscountPercent: null"
6. **Проверяем базу**: должна быть со скидкой 30% (НЕ 50%)

### Шаг 3: Проверка что настройки не переносятся

1. Открываем конструктор для ВТБ
2. Добавляем доп. риски
3. **НЕ ЗАКРЫВАЕМ конструктор**
4. Делаем расчет для Дом.РФ
5. **Проверяем**: конструктор закрылся автоматически
6. Открываем конструктор для Дом.РФ
7. **Проверяем**: доп. риски от ВТБ НЕ перенеслись

## Ключевые моменты

### Когда конструктор закрывается

1. ✅ При нажатии кнопки "Закрыть"
2. ✅ При клике вне модального окна
3. ✅ **При новом расчете (автоматически)** ← НОВОЕ!

### Почему это важно

Если конструктор остается открытым:
- Он продолжает работать со старым контекстом
- При новом расчете контекст обновляется, но конструктор НЕ перерисовывается
- Настройки от старого банка "прилипают" к новому расчету
- Пользователь видит неправильные данные

### Правильный workflow

1. Делаем расчет для банка A
2. Открываем конструктор → меняем настройки
3. Делаем расчет для банка B
4. **Конструктор автоматически закрывается**
5. Открываем конструктор заново → видим правильные настройки для банка B

## Если проблема сохраняется

Убедитесь что:
1. Версия расширения `1.1.0`
2. Конструктор закрывается при новом расчете
3. В консоли нет ошибок
4. Вы НЕ открываете конструктор ДО того как сделали новый расчет
