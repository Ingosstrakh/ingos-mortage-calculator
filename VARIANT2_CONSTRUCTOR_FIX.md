# Исправление конструктора варианта 2

## Проблема
Конструктор варианта 2 "ломался" при изменении:
- Скидки для Сбербанка (вместо 30% другое значение)
- Сумм дополнительных рисков

## Причины

### 1. Базовые премии не пересчитывались
**Было:**
```javascript
const baseNow = ctx.variant2Meta.base || computeVariant2BasePremiums(...);
```

Если `ctx.variant2Meta.base` уже существовал, он не пересчитывался при изменении скидки.

**Исправлено:**
```javascript
const baseNow = computeVariant2BasePremiums(ctx.parsedData, ctx.bankConfig, ins, state.discountPercent);
```

Теперь базовые премии ВСЕГДА пересчитываются при изменении параметров.

### 2. Неправильное использование премии при пересчете скидки
**Было:**
```javascript
const basePrem = Number(b.premiumWithoutDiscount ?? b.premium) || 0;
```

Поле `premiumWithoutDiscount` не существует в объекте borrower.

**Структура borrower:**
```javascript
{
  gender: 'male',
  age: 40,
  share: 1,
  premium: 1000,              // БЕЗ скидки
  premiumWithDiscount: 700    // СО скидкой 30%
}
```

**Исправлено:**
```javascript
const basePrem = Number(b.premium) || 0;  // premium - это уже премия БЕЗ скидки
```

## Результат

✅ Конструктор теперь правильно работает при:
- Изменении скидки для Сбербанка (0-50%)
- Изменении страховой суммы
- Включении/выключении дополнительных рисков
- Изменении сумм дополнительных рисков

## Тестовый кейс
```
сбер 3314955 жен.13.06.1983 кв-ра титул
```

Теперь можно:
- Изменить скидку с 30% на любое значение (0-50%)
- Изменить суммы отделки, движимого имущества, ГО
- Включить/выключить любые доп. риски

Все будет пересчитываться корректно!
