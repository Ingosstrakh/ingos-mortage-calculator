# Исправление скидок в конструкторе варианта 2

## Проблема
Конструктор варианта 2 применял неправильные скидки для банков кроме Сбербанка:
- Вместо фиксированной скидки 30% применялась большая скидка
- Это приводило к "поломке" конструктора

## Решение

### Логика скидок по банкам:

**Сбербанк:**
- Скидка регулируемая: от 0% до 50%
- Пользователь может менять в конструкторе
- Применяется к имуществу, жизни и титулу

**Все остальные банки (ВТБ, ДомРФ, и т.д.):**
- Скидка фиксированная: 30%
- Пользователь НЕ может менять
- Поле скидки скрыто в конструкторе

### Реализация

#### 1. В `computeVariant2BasePremiums()`:

```javascript
// ВАЖНО: только для Сбербанка можно менять скидку (0-50%)
// Для всех остальных банков скидка фиксированная 30%
const isSberbank = bankConfig && bankConfig.bankName === 'Сбербанк';
let discountPercent;

if (isSberbank && discountPercentOverride !== null) {
  // Для Сбербанка используем переданную скидку (0-50%)
  discountPercent = clampDiscountPercent(discountPercentOverride);
} else {
  // Для всех остальных банков фиксированная скидка 30%
  discountPercent = null; // null означает использовать стандартную скидку 30%
}

const discountMultiplier = discountPercent === null ? 0.7 : (1 - discountPercent / 100);
```

#### 2. В `openVariant2Constructor()`:

```javascript
const state = ctx.variant2CustomState || {
  insuranceAmount,
  // Только для Сбербанка можно менять скидку (0-50%)
  // Для остальных банков скидка фиксированная 30% (null = использовать стандартную)
  discountPercent: isSberbank ? 30 : null,
  finishEnabled: Boolean(byObjects['отделка и инженерное оборудование']),
  movableEnabled: Boolean(byObjects['движимое имущество']),
  goEnabled: Boolean(byObjects['гражданская ответственность']),
  finishSum: finishDefault,
  movableSum: movableDefault,
  goSum: goDefault
};
```

#### 3. В `refresh()`:

```javascript
if (isSberbank) {
  // Только для Сбербанка можно менять скидку
  state.discountPercent = clampDiscountPercent(discountInput.value);
  if (state.discountPercent === null) state.discountPercent = 30;
  discountInput.value = String(state.discountPercent);
} else {
  // Для всех остальных банков скидка фиксированная 30% (null)
  state.discountPercent = null;
}

// Для не-Сбербанка передаем null, что означает использовать стандартную скидку 30%
const baseNow = computeVariant2BasePremiums(ctx.parsedData, ctx.bankConfig, ins, state.discountPercent);
```

### Значение `discountPercent`:

- `null` = использовать стандартную скидку 30% (для всех банков кроме Сбербанка)
- `0-50` = кастомная скидка (только для Сбербанка)

### Как работает:

1. **При открытии конструктора:**
   - Сбербанк: показывается поле скидки, значение по умолчанию 30%
   - Другие банки: поле скидки скрыто, используется фиксированная 30%

2. **При изменении параметров:**
   - Сбербанк: пересчитывается с новой скидкой (0-50%)
   - Другие банки: всегда используется 30%

3. **При расчете базовых премий:**
   - Если `discountPercent === null`: используется `discountMultiplier = 0.7` (30% скидка)
   - Если `discountPercent` число: используется `discountMultiplier = 1 - discountPercent / 100`

## Результат

✅ Для Сбербанка: можно менять скидку от 0% до 50%
✅ Для всех остальных банков: фиксированная скидка 30%
✅ Конструктор работает корректно для всех банков
✅ Нет "поломок" при изменении параметров
