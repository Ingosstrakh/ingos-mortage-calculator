# Кнопка "Очистить предыдущие" в конструкторе

## Новая функциональность

Добавлена кнопка **"🗑️ Очистить предыдущие"** в конструкторе варианта 2, которая удаляет все старые результаты расчетов со страницы, оставляя только последний.

## Проблема которую решает

Когда вы делаете несколько расчетов подряд (Сбербанк → ВТБ → Дом.РФ), на странице накапливаются ВСЕ результаты. У каждого результата есть своя кнопка "Конструктор варианта 2". Если случайно нажать на кнопку от СТАРОГО результата, открывается конструктор со старыми данными и настройками.

**Пример проблемы**:
1. Рассчитали Сбербанк → установили скидку 50% в конструкторе
2. Рассчитали ВТБ (результат Сбербанка остался на странице выше)
3. Случайно нажали кнопку "Конструктор" у СТАРОГО результата Сбербанка
4. Открылся конструктор со скидкой 50% (от Сбербанка)
5. Путаница и неправильные данные

## Решение

Кнопка **"Очистить предыдущие"** удаляет все старые результаты, оставляя только последний расчет. Это предотвращает случайное открытие конструктора для старых данных.

## Где находится кнопка

В конструкторе варианта 2, внизу слева:

```
[🗑️ Очистить предыдущие]  [Сбросить]  [Применить к расчету]
```

## Как работает

1. Нажимаете кнопку "🗑️ Очистить предыдущие"
2. Появляется подтверждение: "Очистить все предыдущие результаты расчетов?"
3. Нажимаете "ОК"
4. Все старые результаты удаляются со страницы
5. Остается только последний расчет
6. Конструктор автоматически закрывается
7. Появляется сообщение "✓ Предыдущие результаты очищены"

## Что очищается

### На обычной странице
- Удаляются все результаты из `<div id="result">` кроме последнего

### В расширении
- Удаляются все элементы истории кроме последнего
- Очищается список `.history-item`

## Когда использовать

### Рекомендуется использовать:
1. После нескольких расчетов подряд для разных банков
2. Когда на странице накопилось много результатов
3. Перед важным расчетом, чтобы избежать путаницы
4. Если случайно открыли конструктор для старого результата

### Не обязательно использовать:
1. Если делаете только один расчет
2. Если нужно сравнить результаты разных банков
3. Если уверены что не перепутаете результаты

## Workflow

### Правильный workflow (с очисткой)

1. Рассчитали Сбербанк → открыли конструктор → настроили
2. Рассчитали ВТБ
3. **Открыли конструктор → нажали "Очистить предыдущие"**
4. Теперь на странице только ВТБ
5. Рассчитали Дом.РФ
6. **Снова очистили предыдущие**
7. На странице только Дом.РФ

### Альтернативный workflow (без очистки)

1. Рассчитали Сбербанк
2. Рассчитали ВТБ
3. Рассчитали Дом.РФ
4. На странице ВСЕ три результата
5. **Внимательно смотрите какую кнопку "Конструктор" нажимаете!**

## Технические детали

### Функция очистки

```javascript
window.clearPreviousResults = function() {
  // Для обычной страницы
  const resultDiv = document.getElementById('result');
  if (resultDiv) {
    const lastResult = resultDiv.lastElementChild;
    if (lastResult) {
      resultDiv.innerHTML = '';
      resultDiv.appendChild(lastResult);
    }
  }
  
  // Для расширения
  const historyContainer = document.getElementById('history-container');
  if (historyContainer) {
    const historyItems = historyContainer.querySelectorAll('.history-item');
    if (historyItems.length > 1) {
      for (let i = 0; i < historyItems.length - 1; i++) {
        historyItems[i].remove();
      }
    }
  }
  
  // Закрываем конструктор
  window.closeVariant2Constructor();
  
  alert('✓ Предыдущие результаты очищены');
};
```

### Кнопка в конструкторе

```html
<button type="button" id="variant2-clear-btn" 
  style="border:1px solid #dc3545; background:#fff; color:#dc3545; ...">
  🗑️ Очистить предыдущие
</button>
```

### Обработчик

```javascript
modal.querySelector('#variant2-clear-btn').addEventListener('click', () => {
  if (confirm('Очистить все предыдущие результаты расчетов?')) {
    window.clearPreviousResults();
  }
});
```

2. Сделайте несколько расчетов подряд
3. Откройте конструктор
4. Нажмите кнопку "🗑️ Очистить предыдущие"
5. Подтвердите очистку
6. Проверьте что старые результаты удалены

## Преимущества

1. ✅ Предотвращает случайное открытие конструктора для старых данных
2. ✅ Очищает страницу от лишних результатов
3. ✅ Уменьшает путаницу при работе с несколькими банками
4. ✅ Простой и понятный интерфейс
5. ✅ Подтверждение перед удалением
6. ✅ Работает и на обычной странице, и в расширении

## Примечания

- Кнопка НЕ удаляет текущий (последний) результат
- Кнопка НЕ очищает контекст `window.__LAST_VARIANT2_CONTEXT__`
- Кнопка автоматически закрывает конструктор после очистки
- Очистка необратима (нельзя отменить)
