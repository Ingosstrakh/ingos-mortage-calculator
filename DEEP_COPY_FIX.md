# Исправление: Глубокая копия контекста

## Проблема

Даже после привязки контекста к кнопке, проблема сохранялась. Из логов видно:

```
1. Открывается конструктор для Сбербанка: isSberbank: true ✅
2. Несколько вызовов refresh()
3. Внезапно: isSberbank: false ❌
4. База пересчитывается с неправильными данными
```

**Причина**: Контекст передается ПО ССЫЛКЕ, а не по значению. Когда функция `refresh()` изменяет `ctx.variant2CustomState`, это изменение сохраняется в оригинальном объекте контекста. При следующем открытии конструктора (даже для другого результата), он видит измененный контекст.

### Пример проблемы:

```javascript
// Создаем контекст для ВТБ
window['__VARIANT2_CONTEXT_ctx_1000'] = {
  bankConfig: { bankName: 'ВТБ' },
  variant2CustomState: null
};

// Открываем конструктор
const ctx = window['__VARIANT2_CONTEXT_ctx_1000']; // ❌ Ссылка на объект!

// Внутри refresh() изменяем state
ctx.variant2CustomState = { bankName: 'ВТБ', discountPercent: null };

// Теперь оригинальный объект тоже изменен!
console.log(window['__VARIANT2_CONTEXT_ctx_1000'].variant2CustomState);
// { bankName: 'ВТБ', discountPercent: null }

// При следующем открытии конструктора для ВТБ
// он видит старый state и может перепутать с другим банком
```

## Решение

Делаем ГЛУБОКУЮ КОПИЮ контекста при открытии конструктора. Это гарантирует что изменения внутри конструктора не влияют на оригинальный контекст.

### Было (неправильно):
```javascript
window.openVariant2Constructor = function(forceContext = null) {
  const ctx = forceContext || window.__LAST_VARIANT2_CONTEXT__; // ❌ Ссылка!
  
  // Изменения ctx влияют на оригинальный объект
  ctx.variant2CustomState = { ... };
}
```

### Стало (правильно):
```javascript
window.openVariant2Constructor = function(forceContext = null) {
  let sourceContext = forceContext || window.__LAST_VARIANT2_CONTEXT__;
  
  // ✅ Глубокая копия - изменения не влияют на оригинал
  const ctx = JSON.parse(JSON.stringify(sourceContext));
  
  // Изменения ctx НЕ влияют на оригинальный объект
  ctx.variant2CustomState = { ... };
}
```

## Как это работает

### 1. Без глубокой копии (проблема)
```javascript
// Контекст ВТБ
const vtbContext = {
  bankConfig: { bankName: 'ВТБ' },
  variant2CustomState: null
};

// Открываем конструктор
const ctx = vtbContext; // Ссылка на тот же объект

// Изменяем state
ctx.variant2CustomState = { bankName: 'ВТБ', discountPercent: null };

// Оригинальный объект тоже изменен!
console.log(vtbContext.variant2CustomState); // { bankName: 'ВТБ', ... }
```

### 2. С глубокой копией (решение)
```javascript
// Контекст ВТБ
const vtbContext = {
  bankConfig: { bankName: 'ВТБ' },
  variant2CustomState: null
};

// Открываем конструктор
const ctx = JSON.parse(JSON.stringify(vtbContext)); // Новый объект

// Изменяем state
ctx.variant2CustomState = { bankName: 'ВТБ', discountPercent: null };

// Оригинальный объект НЕ изменен!
console.log(vtbContext.variant2CustomState); // null ✅
```

## Преимущества

### ✅ Изоляция изменений
Изменения в конструкторе не влияют на оригинальный контекст, сохраненный в кнопке.

### ✅ Независимые сессии
Можно открывать конструктор для разных результатов, и они не будут влиять друг на друга.

### ✅ Предсказуемое поведение
Каждое открытие конструктора начинается с чистого состояния (или сохраненного в контексте).

### ✅ Защита от мутаций
Невозможно случайно изменить контекст другого результата.

## Технические детали

### JSON.parse(JSON.stringify())
Это простой способ сделать глубокую копию объекта в JavaScript:

```javascript
const original = { a: 1, b: { c: 2 } };
const copy = JSON.parse(JSON.stringify(original));

copy.b.c = 3;
console.log(original.b.c); // 2 (не изменился)
```

**Ограничения:**
- Не копирует функции
- Не копирует undefined
- Не копирует Symbol
- Не копирует циклические ссылки

**Для нашего случая это идеально**, т.к. контекст содержит только данные (объекты, числа, строки, массивы).

### Альтернативы

#### structuredClone (современный способ)
```javascript
const ctx = structuredClone(sourceContext);
```
Но не поддерживается в старых браузерах.

#### Lodash cloneDeep
```javascript
const ctx = _.cloneDeep(sourceContext);
```
Требует подключения библиотеки.

#### Ручное копирование
```javascript
const ctx = {
  parsedData: { ...sourceContext.parsedData },
  bankConfig: { ...sourceContext.bankConfig },
  // ... и т.д.
};
```
Слишком многословно и легко забыть поле.

## Workflow

### Сценарий 1: Открытие конструктора для ВТБ
```
1. Нажали кнопку "Конструктор" у ВТБ
2. Функция получает контекст ВТБ
3. Делается глубокая копия контекста
4. Работа с копией (изменения state, base и т.д.)
5. Оригинальный контекст ВТБ не изменен
```

### Сценарий 2: Открытие для Сбербанка после ВТБ
```
1. Нажали кнопку "Конструктор" у Сбербанка
2. Функция получает контекст Сбербанка (не ВТБ!)
3. Делается глубокая копия контекста Сбербанка
4. Работа с копией (скидка 30-50% для Сбербанка)
5. Контекст ВТБ не затронут
```

### Сценарий 3: Повторное открытие для ВТБ
```
1. Нажали кнопку "Конструктор" у ВТБ снова
2. Функция получает оригинальный контекст ВТБ
3. Делается новая глубокая копия
4. Начинаем с чистого состояния (или сохраненного в контексте)
5. Предыдущие изменения не влияют
```

## Заключение

Глубокая копия контекста гарантирует что каждое открытие конструктора работает с изолированными данными. Изменения в одном конструкторе не влияют на другие результаты.
